<h3>Character I/O using <code>get( )</code>, <code>put(
)</code> and <code>putback(
)</code></h3>
<p>
Our use of input streams so far has been restricted to reading a single
"word" of data at a time where the start and end of words are indicated
by <b>whitespaces</b>, such as a space, multiple spaces, a
tab, or a
return. A word is a sequence of characters, and sometimes we will need
to read or write a single character at a time---this is possible using
the <code>get()</code> and <code>put()</code>
stream member
functions.
</p>

using single quotes), as shown to the right. </td>

<h3>Checking for the end-of-file using <code>eof()</code></h3>
Care must be taken when reading characters from a file to check if the
end has been reached. Most versions of C++ incorporate an end-of-file
(EOF) flag into the ifstreams, and programmers use the member function "<code>eof()</code>"
for ifstreams to test if this flag is set to <i>TRUE</i>
or <i>FALSE</i>.
<p>If the stream is positioned at the end of a file and the EOF
flag is
FALSE, the statement:
</p>
<p>
</p>
<table>
<tbody>
<tr>
<th width="20"><br>
</th>
<td>
<pre><code> in_stream.get( ch );</code><br></pre>
</td>
</tr>
</tbody>
</table>
will set the EOF flag to TRUE, but unfortunately, will leave the <code>char</code>
variable "<code>ch</code>" in an unpredictable state. <b>It
is
important to always check the EOF flag using <code>in_stream.eof()</code>
before using the result of the <code>get()</code> member
function!</b>
